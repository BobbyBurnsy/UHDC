IT Pro Summary

This is a professional remediation tool designed to instantly resolve deep-rooted web browser corruption. It utilizes SMB to secure a forensic backup of the user's local bookmarks, deploys PsExec to forcefully sever active browser process locks, purges the corrupted AppData profiles, and seamlessly injects the bookmarks back into the newly generated profile directories.

When to Use This Tool

The Scenario

A user submits a ticket stating their Google Chrome or Microsoft Edge keeps crashing upon opening, or a critical internal web app is throwing continuous graphical errors. A standard cache clear doesn't work. The technician knows they need to completely delete the local browser profile, but if they do that manually, the user will lose all their saved bookmarks and be frustrated.

The Fix

The technician ensures both the "Target PC" and "Target User" fields are populated in the UHDC and clicks "Browser Reset." In roughly three seconds, the script reaches across the network, saves the bookmark files, terminates the frozen browser, completely drops the corrupt User Data folder, and restores the bookmarks. The user clicks the Chrome icon on their desktop, the browser opens lightning fast as if it was freshly installed, and all their bookmarks are perfectly intact.

Interactive Training Engine

When a junior technician runs this tool with Training Mode enabled, the UHDC pauses execution at each critical phase. It displays a pop-up explaining why the action is necessary, what the in-person equivalent would be, and reveals the exact PowerShell code being executed under the hood. Here is what the trainee sees during a Browser Reset:

[TRAINING] STEP 1: BACKUP BOOKMARKS

We are copying the user's Bookmarks file from their remote AppData directory to a safe local backup folder.

IN-PERSON EQUIVALENT:
If you were at the user's desk, you would open File Explorer, navigate to %LocalAppData%\Google\Chrome\User Data\Default, copy the 'Bookmarks' file, and paste it to the Desktop for safekeeping.

Copy-Item '\\$Target\c$\Users\$TargetUser\AppData\Local\Google\Chrome\User Data\Default\Bookmarks' -Destination 'C:\UHDC\Backups\BrowserReset_$Target\Chrome_BM' -Force
[TRAINING] STEP 2: TERMINATE BROWSER PROCESSES

We must forcefully terminate any running instances of Chrome or Edge. If the browser is open (or running in the background), Windows will lock the AppData files and prevent us from deleting them.

IN-PERSON EQUIVALENT:
If you were at the user's desk, you would press Ctrl+Shift+Esc to open Task Manager, locate all 'Google Chrome' and 'Microsoft Edge' processes, right-click them, and select 'End task'.

psexec.exe \\$Target -s taskkill /F /IM chrome.exe /IM msedge.exe /T
[TRAINING] STEP 3: PURGE CORRUPTED PROFILES

We are deleting the entire 'User Data' directory for both browsers. This permanently wipes the corrupted cache, cookies, history, and extensions.

IN-PERSON EQUIVALENT:
If you were at the user's desk, you would navigate to %LocalAppData%\Google\Chrome\ and permanently delete the 'User Data' folder. You would then repeat this for Edge.

Remove-Item '\\$Target\c$\Users\$TargetUser\AppData\Local\Google\Chrome\User Data' -Recurse -Force
[TRAINING] STEP 4: RESTORE BOOKMARKS

We are recreating the 'Default' profile directory structure and copying the backed-up Bookmarks file back into place before the user opens the browser again.

IN-PERSON EQUIVALENT:
If you were at the user's desk, you would open Chrome once to let it automatically recreate the default folders, close the browser, and then copy the saved 'Bookmarks' file from the Desktop back into the 'Default' folder.

New-Item -ItemType Directory -Path '\\$Target\c$\Users\$TargetUser\AppData\Local\Google\Chrome\User Data\Default' -Force
Copy-Item 'C:\UHDC\Backups\BrowserReset_$Target\Chrome_BM' '\\$Target\...\Default\Bookmarks' -Force

Technical Execution Pipeline

  1. Data Extraction (SMB): Bypasses WinRM entirely. Connects to the endpoint via Port 445 (SMB) using the hidden C$ admin share. It navigates explicitly to \Users\$TargetUser\AppData\Local\... and copies the SQLite `Bookmarks` files to a safe local backup directory.
  2. Process Termination (RPC/PsExec): Deploys PsExec.exe over Port 135 (RPC) and Port 445 (SMB) to run a remote taskkill /F /IM chrome.exe /T command under the NT AUTHORITY\SYSTEM context. This forcefully drops all active memory and file system locks on the remote browser.
  3. Profile Purge (SMB): Re-utilizes the C$ share to execute a Remove-Item -Force -Recurse against the entire \User Data directory for both Edge and Chrome, effectively wiping the corrupted state from the hard drive.
  4. Data Restoration (SMB): Recreates the core directory structure and injects the saved `Bookmarks` files back into the default profile location before the user ever reopens the application.
  5. Audit Logging: Logs the successful execution to the central UHDC CSV database.

Q&A

Q: Why do we use PsExec to kill the browser processes instead of a standard WinRM command?
If the user is actively staring at a frozen browser, the processes are locked tightly in their user session. Attempting to kill it via WMI or standard remote PowerShell can sometimes fail or throw access denied errors depending on the execution context. PsExec runs as the SYSTEM account, which gives it absolute authority to terminate the process tree, bypassing all session isolation restrictions.
Q: Why does this script require the technician to enter BOTH the Target PC and the Username?
Because it leverages direct SMB file paths (\\TargetPC\C$\Users\Username\AppData...) instead of querying the environmental variables of the remote machine. It needs the specific AD username to build the correct network path to extract the bookmarks.
Q: Will the user lose their saved passwords or browser extensions?
Yes, any locally cached data outside of bookmarks (like history, extensions, and local passwords) is purged. However, if the user is signed into Google Sync or Microsoft Enterprise Sync, their extensions and passwords will automatically pull back down from the cloud the moment they reopen the browser. This script specifically rescues the local bookmarks for users who do not utilize cloud sync.
Q: Can this be run while the user is actively working?
It can, but the user will see their browser suddenly vanish from their screen. It is best practice to send them a quick "Net Send" message via the console or call them to say, "I am going to quickly reset your browser now" before clicking the button.
