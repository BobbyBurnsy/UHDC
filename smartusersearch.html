IT Pro Summary

This is the core data aggregation engine for the Unified Help Desk Console. It acts as the "tip of the spear" for live phone support. It executes a dual-pronged query—pulling real-time identity and security metrics directly from Active Directory, while simultaneously parsing the local UserHistory.json database to instantly map the user to their physical hardware.

When to Use This Tool

The Scenario

A user calls the help desk complaining that they "can't log into the VPN" or "can't access their email." A Tier 1 technician might immediately start trying to reinstall the VPN client, completely missing the root cause because looking up their full security profile in standard AD tools is tedious. Furthermore, the user is working from a shared conference room today and has no idea what the computer name is, stalling the troubleshooting process.

The Fix

The technician types the user's name into the console and hits enter. SmartUserSearch.ps1 instantly cross-references their AD profile. The console immediately flashes a high-visibility text alert: !!! EXPIRED (2 days ago) !!! and shows YES (LOCKED).

The technician instantly knows the issue isn't the VPN client; the user's password expired over the weekend, and they locked themselves out trying to guess it. Furthermore, the script automatically lists the last 3 computers the user logged into. It takes the most recent one (LAPTOP-US-4829) and magically auto-fills it into the "Target PC" box in the console UI. The technician is instantly ready to push a remote GPUpdate or unlock the account without ever asking the user for their hostname.

[TRAINING] STEP 1: CORRELATE USER TO COMPUTER

WHEN TO USE THIS:
This is the first step in any remote support scenario. A user calls in with an issue, but they don't know their computer name, making it impossible to remote in or push fixes.

WHAT IT DOES:
We are parsing the central 'UserHistory.json' database. We check if your input matches a known Username. If it does, we pull a list of the most recent PCs they logged into. We then use a special GUI tag '[GUI:UPDATE_TARGET...]' to automatically fill the 'Target PC' boxes on the right side of the console so you are instantly ready to work.

IN-PERSON EQUIVALENT:
Asking the user, 'Can you read me the asset tag sticker on the bottom of your laptop?' or having them open the Start Menu, type 'cmd', and run the 'hostname' command.

$raw = Get-Content $HistoryFile -Raw | ConvertFrom-Json
$userHistory = $raw | Where-Object { $_.User -eq $TargetUser }
Write-Host "[GUI:UPDATE_TARGET:$($userHistory[0].Computer)]"
[TRAINING] STEP 2: QUERY AD & CALCULATE PASSWORD EXPIRATION

WHEN TO USE THIS:
Use this to instantly verify if an account is locked out, disabled, or if their password is about to expire. It also checks if they have the correct security groups (like VPN or Admin access).

WHAT IT DOES:
We use 'Get-ADUser' to pull the account details. Then, we do some math: We query the domain's default password policy ('Get-ADDefaultDomainPasswordPolicy') to find the 'MaxPasswordAge' (e.g., 90 days). We add those 90 days to the user's 'PasswordLastSet' date, and subtract today's date to tell you exactly how many days they have left before they are locked out.

IN-PERSON EQUIVALENT:
Opening Active Directory Users and Computers (ADUC), searching for the user, checking the 'Account' tab to see if the 'Unlock account' box is checked, and clicking the 'Member Of' tab to read through their assigned groups.

$adObj = Get-ADUser -Identity $TargetUser -Properties LockedOut, PasswordLastSet, MemberOf
$policy = Get-ADDefaultDomainPasswordPolicy
$expDate = $adObj.PasswordLastSet.AddDays($policy.MaxPasswordAge.Days)

Technical Execution Pipeline

  1. JSON Database Parsing (SMB): Bypasses Active Directory entirely for the hardware lookup. It connects to the network share over Port 445 (SMB), validates the health of the UserHistory.json file, and deserializes the entire object array into memory.
  2. Identity Correlation: Iterates through the JSON array, filtering exclusively for records matching the provided Target User string. It utilizes a hashtable logic to deduplicate the results, ensuring it outputs a clean list of unique workstations the user has logged into, ranked by timestamp.
  3. Active Directory Query (LDAP): Executes a targeted Get-ADUser query over Port 389 (LDAP) or Port 636 (LDAPS), pulling an expanded property set (Office, Title, Department, LockedOut, PasswordLastSet, MemberOf).
  4. Security & Expiry Calculation: Automatically queries the domain's default password policy to dynamically calculate the maximum password age. It calculates the days left and triggers high-visibility UI alerts if the account is expiring or expired.
  5. Group Truncation & UI Handoff: Filters the user's groups against a RegEx keywords string (e.g., M365|VPN|Admin) to only display high-value security groups in the console HUD. Finally, it outputs a hidden [GUI:UPDATE_TARGET:...] string that the frontend catches to automatically populate the "Target PC" text box.

Q&A

Q: What network ports does this script require?
The script requires Port 445 (SMB) to read the UserHistory.json map from the network share. It relies on Port 389/636 (LDAP) to query Active Directory for the user's profile and the domain's default password policy.
Q: If a user belongs to 50 different Active Directory groups, will this script flood the console UI with all of them?
No, by design. The script uses a specific regex filter to only display critical access groups (like VPN, Admin, or specific software licenses). It explicitly hides standard domain groups to keep the technician's Heads Up Display clean, noting at the bottom: ...plus [X] other standard groups.
Q: Where does the "Days until expiry" number come from?
It is dynamically calculated. The script queries the domain's default password policy to find the exact "Maximum Password Age" (e.g., 90 days). It then takes the user's PasswordLastSet timestamp, adds 90 days to it, and subtracts the current date. This guarantees the expiration warning is always perfectly accurate.
Q: What does the [GUI:UPDATE_TARGET:...] text at the bottom do?
It is a bridge between the PowerShell backend and the graphical frontend. The script outputs the most recent computer the user logged into in a specific bracketed format. The console's underlying C# code is listening for that string; when it sees it, it automatically takes the computer name and pastes it into the "Target PC" box in the UI, saving the technician from having to type or copy/paste it manually.
