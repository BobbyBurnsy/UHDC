IT Pro Summary

This is an automated hardware diagnostic and routing tool. It queries the target computer's WMI/CIM repository to extract the embedded motherboard serial number and system manufacturer. It then utilizes dynamic string matching to automatically construct the correct vendor support URL (Dell, Lenovo, HP, or Microsoft) and launches the technician's local web browser directly to the device's specific warranty entitlement page.

When to Use This Tool

The Scenario

A user calls the help desk reporting physical hardware damage—perhaps they dropped their laptop and cracked the screen, or the keyboard has stopped working. Before you can order replacement parts or dispatch a field technician, you need to verify if the device is still covered under the manufacturer's active warranty or an accidental damage protection plan. Asking the user to flip the laptop over and read a tiny, faded 7-to-10 character alphanumeric string over the phone almost always results in typos and frustration.

The Fix

While the user is explaining the damage, the technician clicks "Smart Warranty" in the UHDC. The script silently reaches across the network, queries the motherboard, identifies that it is a Lenovo ThinkPad, grabs the serial number, and instantly opens the Lenovo Support portal on the technician's screen. The technician can immediately see the warranty expiration date and tell the user, "Good news, you have Next Business Day Onsite coverage. I'll get a tech dispatched to your desk tomorrow."

Interactive Training Engine

When a junior technician runs this tool with Training Mode enabled, the UHDC pauses execution at each critical phase. It displays a pop-up explaining why the action is necessary, what the in-person equivalent would be, and reveals the exact PowerShell code being executed under the hood. [TRAINING] STEP 1: QUERY HARDWARE INFORMATION

WHEN TO USE THIS:
Use this when a user reports physical hardware damage (e.g., cracked screen, failing drive, swollen battery) or when preparing to order replacement parts and you need to verify if the device is still covered under the manufacturer's active warranty or accidental damage protection plan.

WHAT IT DOES:
We are establishing a remote WMI/CIM session to query the target computer's motherboard (Win32_BIOS) for its embedded Serial Number/Service Tag, and the system enclosure (Win32_ComputerSystem) for the Manufacturer (Make).

IN-PERSON EQUIVALENT:
If you were physically at the user's desk, you would flip the laptop over and read the tiny printed sticker on the bottom chassis, or open an elevated Command Prompt and type 'wmic bios get serialnumber' followed by 'wmic computersystem get manufacturer'.

$bios = Get-CimInstance -ComputerName $Target -ClassName Win32_BIOS
$cs = Get-CimInstance -ComputerName $Target -ClassName Win32_ComputerSystem
[TRAINING] STEP 2: LAUNCH VENDOR WARRANTY PORTAL

WHEN TO USE THIS:
Immediately after retrieving the serial number to check the exact expiration date and entitlement level (e.g., Next Business Day Onsite vs. Depot Repair).

WHAT IT DOES:
We are using the retrieved Manufacturer name to automatically determine the correct vendor support portal (Dell, Lenovo, HP, or Microsoft). We then inject the Serial Number directly into the URL and launch it in your local default web browser.

IN-PERSON EQUIVALENT:
You would open a web browser, search for 'Dell Warranty Check' (or the respective vendor), navigate to their support page, manually type the alphanumeric serial number into the search box, and click Submit.

Start-Process "$url"

Technical Execution Pipeline

  1. Pre-Execution Validation: Executes a fast ICMP Ping to verify endpoint connectivity before attempting WMI connections.
  2. Hardware Telemetry (WMI/CIM): Establishes a remote session over Port 5985 (HTTP) to query two specific WMI classes: Win32_BIOS (to extract the SerialNumber) and Win32_ComputerSystem (to extract the Manufacturer).
  3. Dynamic Routing Logic: Uses a switch or if/match statement to evaluate the manufacturer string (e.g., matching "Hewlett-Packard" to HP). It then concatenates the serial number into the specific REST API or search query URL format required by that vendor's website.
  4. Local Shell Handoff: Executes Start-Process $url, which instructs the technician's local Windows OS to open the constructed link in their default web browser.
  5. Audit Logging: Logs the successful query (including the Make and Serial Number) to the central UHDC CSV database via SMB (Port 445).

Q&A

Q: Which hardware vendors are currently supported by the auto-routing logic?
The script currently supports automatic URL construction for Dell, Lenovo, HP, and Microsoft Surface devices. If a different manufacturer is detected (e.g., Panasonic or a custom-built PC), the script will still display the serial number in the console for manual lookup, but it will not attempt to open a web browser.
Q: Why does the script warn me about Microsoft Surface devices?
Unlike Dell or Lenovo, which allow public, unauthenticated queries against their warranty databases, Microsoft typically requires an administrator to log into the "Microsoft 365 admin center" or "Surface Support Portal" to view warranty entitlements. The script will route you to the correct login page, but it cannot bypass the authentication requirement.
Q: What happens if the WMI query fails?
If the target machine has a corrupted WMI repository or a strict firewall blocking Port 5985, the script will fail gracefully. It will output an error to the console suggesting "Possible cause: Firewall, RPC error, or Access Denied," allowing the technician to pivot to a different troubleshooting method.
