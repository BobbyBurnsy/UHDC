IT Pro Summary

This is a rapid diagnostic tool that enforces the "trust but verify" rule of IT support. It queries the remote computer's WMI/CIM repository for the Win32_OperatingSystem class to extract the exact LastBootUpTime. It calculates the precise timespan the machine has been running continuously and generates a high-visibility alert if the endpoint has not been rebooted in over 7 days.

When to Use This Tool

The Scenario

A user calls the help desk complaining about general PC slowness, bizarre application glitches, or audio devices not working. You ask the user, "Have you tried restarting your computer today?" The user confidently replies, "Yes, I just rebooted it right before I called you."

The Fix

While the user is talking, the technician clicks "Get Uptime" in the UHDC. The script instantly queries the machine and reveals an uptime of 14 Days, 6 Hours. The technician now knows the user is confusing "closing the laptop lid" or "logging off" with a full system restart. Instead of arguing with the user, the technician can politely say, "It looks like Windows didn't fully clear its memory cache on that last restart. Let me send a command to force a deep reboot for you right now."

[TRAINING] STEP 1: QUERY SYSTEM UPTIME

WHEN TO USE THIS:
Use this when a user complains about general PC slowness, bizarre application glitches, or when they claim they 'just rebooted' but the issue persists. Users frequently confuse turning off the monitor, closing a laptop lid, or logging off with a full system restart.

WHAT IT DOES:
We are establishing a remote WMI/CIM session to query the 'Win32_OperatingSystem' class. We retrieve the exact 'LastBootUpTime' property and subtract it from the current time to calculate the precise number of days, hours, and minutes the machine has been running continuously.

IN-PERSON EQUIVALENT:
If you were physically at the user's desk, you would press Ctrl+Shift+Esc to open Task Manager, click the 'Performance' tab, select 'CPU', and look at the 'Up time' counter at the bottom. Alternatively, you could open an elevated Command Prompt and type 'systeminfo | find "System Boot Time"'.

$os = Get-CimInstance -ComputerName $Target -ClassName Win32_OperatingSystem
$boot = $os.LastBootUpTime
$uptime = (Get-Date) - $boot

Technical Execution Pipeline

  1. Pre-Execution Validation: Executes a fast ICMP Ping to verify endpoint connectivity before attempting WMI connections.
  2. Telemetry Extraction (WMI/CIM): Establishes a remote session over Port 5985 (HTTP) using the modern Get-CimInstance cmdlet to query the Win32_OperatingSystem class.
  3. Timespan Calculation: Extracts the LastBootUpTime property (which CIM automatically converts from a WMI datetime string into a standard PowerShell [datetime] object) and subtracts it from Get-Date to create a [timespan] object.
  4. Logic & Alerting: Evaluates the $uptime.Days property. If the value is greater than 7, it injects a yellow warning string into the console output to alert the technician of potential system instability.
  5. Audit Logging: Logs the successful query and the calculated uptime days to the central UHDC CSV database via SMB (Port 445).

Q&A

Q: Why do users so frequently think they rebooted when they actually didn't?
Aside from simply confusing "logging off" with "restarting," the biggest culprit is the Windows "Fast Startup" feature. By default, when a user clicks "Shut down" in Windows 10/11, the PC doesn't actually turn off completely; it logs the user out and hibernates the Windows kernel to make the next boot faster. Because the kernel never stopped running, the WMI Uptime counter never resets. Only a true "Restart" command (or a forced shutdown) resets the counter.
Q: Why does the script use `Get-CimInstance` instead of the older `Get-WmiObject`?
Get-WmiObject relies on legacy DCOM/RPC protocols (Port 135 and dynamic high ports), which are frequently blocked by modern endpoint firewalls. Get-CimInstance is the modern standard; it routes the exact same WMI queries over the WinRM protocol (Port 5985/HTTP), which is much more firewall-friendly and reliable across enterprise networks.
