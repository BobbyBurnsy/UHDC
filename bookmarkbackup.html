IT Pro Summary

This is a targeted data preservation tool designed to act as a safety net before performing destructive administrative actions. It bypasses WinRM entirely, utilizing the SMB protocol to connect to the remote endpoint's hidden C$ administrative share. It extracts the raw SQLite bookmark files for both Google Chrome and Microsoft Edge, saving them to a timestamped local directory for safekeeping.

When to Use This Tool

The Scenario

You are on a call with a user whose Windows profile has become completely corrupted, or they are scheduled to receive a hardware replacement (PC swap) today. The user is panicking because they have hundreds of critical, work-related bookmarks saved locally to their browser, and they are not signed into a cloud-sync account (like an Enterprise Microsoft or Google account).

The Fix

Before you delete their profile or tell them to shut down their old PC, you click "Bkmk Backup" in the UHDC. The script instantly reaches across the network, grabs the bookmark files, and saves them to your local C:\UHDC\Bookmarks folder. You can now confidently proceed with the profile rebuild or PC swap, knowing you have a secure copy of their data ready to inject into their new setup.

Technical Execution Pipeline

  1. Pre-Execution Validation: Executes a fast ICMP Ping to verify endpoint connectivity. If the AD Username box is blank, it spawns a GUI input box to request the target user's name.
  2. Directory Generation: Creates a local, timestamped destination folder (e.g., C:\UHDC\Bookmarks\LAPTOP-US-4829-jsmith-20260225-1422) to ensure backups are never overwritten.
  3. SMB Extraction: Connects to the remote endpoint via Port 445 (SMB) using the C$ share. It explicitly targets the AppData\Local paths for both Chrome and Edge, copying the files locally.
  4. Shell Handoff: Executes explorer.exe with the /select argument, breaking out of the PowerShell runspace to visually highlight the newly created backup folder for the technician.
  5. Audit Logging: Logs the successful extraction to the central UHDC CSV database.

Q&A

Q: Why use SMB instead of WinRM for this tool?
While WinRM is excellent for executing commands, transferring files over WinRM can be slow and memory-intensive because the data has to be serialized and deserialized through the HTTP pipeline. SMB (Port 445) is the native Windows file-sharing protocol, making it significantly faster and more reliable for raw data extraction.
Q: Does this backup the user's saved passwords or browsing history?
No. It only backs up the Bookmarks file. Saved passwords in Chromium browsers are encrypted using the user's unique Windows DPAPI (Data Protection API) key. You cannot simply copy a password file from one PC to another; it will be unreadable. If a user needs their passwords transferred, they must sign into a browser sync profile.
Q: What does the `/select` argument do in the final step?
When you run explorer.exe "C:\Folder", Windows opens that folder. When you run explorer.exe /select,"C:\Folder", Windows opens the parent directory (the C: drive) and automatically highlights the specific folder. This is a quality-of-life feature that saves the technician from having to hunt for the newly created backup among dozens of other files.
