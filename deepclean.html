IT Pro Summary

This tool is a comprehensive remote endpoint cleanup script. It executes a synchronized local and remote pipeline to recover storage space and resolve stale cache issues without disrupting the active user. It handles system-level cache rebuilds, recursive temp file deletion, and dispatches native Windows cleanup utilities asynchronously.

When to Use This Tool

The Scenario

A user calls the help desk complaining that their computer is running incredibly slow, or a new software deployment via MECM/SCCM is repeatedly failing with a "Hash Mismatch" or "Insufficient Disk Space" error.

The Fix

The technician clicks "Deep Clean." The script immediately purges and rebuilds the corrupted SCCM cache so the software can redownload cleanly. It then wipes out gigabytes of bloated system temp files, user temp files, and empties the Recycle Bin. Finally, it triggers a silent Windows Disk Cleanup job in the background to handle legacy Windows Update payloads. It resolves the SCCM deployment failure and restores performance to the endpoint in a single click.

[TRAINING] STEP 1: PURGE MECM (SCCM) CACHE

WHEN TO USE THIS:
Use this when Software Center deployments are stuck at 0%, failing with 'hash mismatch' errors, or when the C: drive is critically low on space due to large cached application installers.

WHAT IT DOES:
We are using PsExec (running as the SYSTEM account) to forcefully delete and recreate the 'ccmcache' directory, bypassing standard permission blocks.

IN-PERSON EQUIVALENT:
Open Control Panel > Configuration Manager > Cache tab > Configure Settings > Delete Files.

psexec.exe \\$Target -s cmd /c "rd /s /q C:\Windows\ccmcache & mkdir C:\Windows\ccmcache"
[TRAINING] STEP 2: CLEAR TEMP FOLDERS & RECYCLE BIN

WHEN TO USE THIS:
Use this when a user is experiencing bizarre application glitches (like Office apps crashing), cannot open certain files, or needs immediate disk space recovery.

WHAT IT DOES:
We are establishing a WinRM session to recursively delete files in 'C:\Windows\Temp', iterate through every user profile to clear their 'AppData\Local\Temp' folders, and forcefully empty the Recycle Bin.

IN-PERSON EQUIVALENT:
Press Win+R, type '%temp%', select all files, and delete. Repeat for 'C:\Windows\Temp'. Finally, right-click the Recycle Bin on the desktop and select 'Empty Recycle Bin'.

Invoke-Command -ComputerName $Target -ScriptBlock {
    Remove-Item -Path 'C:\Windows\Temp\*' -Recurse -Force
    Remove-Item -Path 'C:\Users\*\AppData\Local\Temp\*' -Recurse -Force
    Clear-RecycleBin -Force
}
[TRAINING] STEP 3: TRIGGER BACKGROUND DISK CLEANUP

WHEN TO USE THIS:
Use this for general PC maintenance, post-Windows Update cleanup, or to safely remove old Windows installation files (Windows.old) without interrupting the user.

WHAT IT DOES:
We are remotely triggering the native Windows Disk Cleanup utility ('cleanmgr.exe') using the '/sagerun:1' flag, which runs a pre-configured, silent deep clean in the background.

IN-PERSON EQUIVALENT:
Open the Start Menu, search for 'Disk Cleanup', select the C: drive, click 'Clean up system files', check all the boxes, and click OK.

Invoke-Command -ComputerName $Target -ScriptBlock { Start-Process -FilePath 'cleanmgr.exe' -ArgumentList '/sagerun:1' -WindowStyle Hidden }

Technical Execution Pipeline

  1. Pre-Execution Validation: Executes a fast ICMP Ping to verify endpoint connectivity before attempting RPC/WinRM connections.
  2. System-Level SCCM Purge: Deploys PsExec over SMB/RPC (Ports 445 & 135) to execute a strict deletion and recreation of C:\Windows\ccmcache running under the NT AUTHORITY\SYSTEM context. This ensures a 100% success rate regardless of local SCCM client health.
  3. WinRM Deep Clean: Establishes a remote PowerShell session (Port 5985) to silently and recursively wipe standard system bloat (C:\Windows\Temp).
  4. Dynamic Profile Traversal: Enumerates the C:\Users directory to wipe AppData Temp folders across all existing user profiles, and forcefully empties the Recycle Bin across all drives.
  5. Asynchronous Disk Cleanup: Triggers a pre-configured cleanmgr.exe /sagerun:1 process over WinRM. By using Start-Process with a hidden window style, it dispatches the job to the background without hanging the active WinRM session.
  6. Audit Logging: Logs the successful execution to the central UHDC CSV database via SMB (Port 445).

Q&A

Q: If a user is actively working and has files open, will the script fail or crash their applications?
Neither. The script uses the -ErrorAction SilentlyContinue parameter across all file deletion loops. If a cache file is actively locked by an open application (like Word or Chrome), PowerShell smoothly skips that specific file and moves to the next one without throwing errors or disrupting the user's workflow.
Q: Why use PsExec to clear the SCCM cache instead of the native COM object?
The COM object relies heavily on a perfectly healthy WMI repository on the target machine. If the client is broken, the cleanup fails. Executing a strict folder deletion and recreation as the SYSTEM account via PsExec bypasses local SCCM client health issues and guarantees the cache is purged.
Q: What does the /sagerun:1 flag do for cleanmgr.exe?
The cleanmgr.exe utility requires a user to manually check boxes in a GUI to tell it what to clean. The /sagerun:1 flag tells the utility to run silently in the background using a pre-configured set of rules (usually configured via /sageset:1 on your master image), allowing it to automatically clean up Windows Update files, memory dumps, and delivery optimization caches without any user interaction.
