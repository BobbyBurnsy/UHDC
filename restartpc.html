IT Pro Summary

This is a GUI-driven power management tool that utilizes PsExec to bypass common WMI and RPC firewall restrictions. It generates a custom WPF interface, allowing the technician to select between graceful reboots, forced hardware shutdowns, and targeted user session logoffs, executing the native Windows commands under the SYSTEM context.

When to Use This Tool

The Scenario

A user's computer is completely frozen, their Start Menu won't open, or a rogue application is preventing them from clicking the reboot button. Alternatively, a user walked away from a shared kiosk machine without logging out, and the next user cannot access the system.

The Fix

The technician clicks "Restart Options" in the UHDC. A clean, dark-themed menu pops up offering multiple choices. If the user is actively on the phone, the tech selects "Standard Restart," which gives the user a 60-second on-screen warning to save their work. If the machine is totally locked up, the tech selects "Force Restart" to instantly drop all processes. If it's a shared kiosk, the tech selects "Force Logoff," which instantly kicks the previous user out and returns the machine to the Ctrl+Alt+Del screen without having to wait for a full hardware reboot.

Interactive Training Engine

When a junior technician runs this tool with Training Mode enabled, the UHDC pauses execution after they make a selection from the GUI menu. It displays a pop-up explaining the specific command they chose. Here is what the trainee sees depending on their selection:

[TRAINING] STEP 1: STANDARD RESTART

WHEN TO USE THIS:
Use this for general troubleshooting when a user is actively working on the PC. It gives them a 60-second warning to save their documents before the computer reboots.

WHAT IT DOES:
We use PsExec to run the native 'shutdown.exe' command on the target. The '/r' flag means restart, and '/t 60' sets a 60-second timer.

psexec.exe \\$Target -s shutdown /r /t 60
[TRAINING] STEP 1: FORCE RESTART

WHEN TO USE THIS:
Use this when a computer is completely frozen, stuck on a black screen, or a rogue application is preventing a standard restart. WARNING: The user will lose any unsaved work.

WHAT IT DOES:
We add the '/f' (force) flag and set the timer to '/t 0'. This tells Windows to instantly kill all running applications without waiting for them to close gracefully.

psexec.exe \\$Target -s shutdown /r /f /t 0
[TRAINING] STEP 1: FORCE LOGOFF

WHEN TO USE THIS:
Use this when a user's profile is locked up, their Start Menu won't open, or you need to kick a disconnected user off a shared workstation without rebooting the entire PC.

WHAT IT DOES:
Instead of 'shutdown.exe', we use PsExec to run 'rwinsta console'. This forcefully terminates the active Windows session (the 'console' session) and returns the PC to the Ctrl+Alt+Del login screen.

psexec.exe \\$Target -s rwinsta console

Technical Execution Pipeline

  1. Pre-Execution Validation: Executes a fast ICMP Ping to verify endpoint connectivity.
  2. Dynamic UI Generation: Uses PowerShell to render a custom WPF XAML window, presenting the technician with a list of clearly defined power actions and descriptions.
  3. Payload Execution (RPC/SMB): Based on the selection, it dispatches PsExec.exe to the remote endpoint over Port 135 (RPC) and Port 445 (SMB). It executes the native Windows binary (shutdown.exe or rwinsta.exe) under the NT AUTHORITY\SYSTEM context.
  4. Audit Logging: Logs the specific power action chosen (e.g., "Power Control Executed: ForceRestart") to the central UHDC CSV database.

Q&A

Q: Why use PsExec instead of the native `Restart-Computer` PowerShell cmdlet?
The native Restart-Computer cmdlet relies heavily on WMI (Windows Management Instrumentation). If a computer is severely frozen, the WMI service is often unresponsive, causing the command to fail. By using PsExec, we bypass WMI entirely and interact directly with the Service Control Manager to execute the native shutdown.exe binary, which is significantly more reliable on locked-up machines.
Q: What exactly is `rwinsta`?
rwinsta stands for "Reset Windows Station." It is a legacy Terminal Services command. By passing the argument console, we are telling the operating system to instantly drop the physical console session (the user sitting at the keyboard). It is much faster than a full reboot because it leaves the Windows kernel and background services running while simply resetting the user interface.
Q: If I accidentally click "Standard Restart" on the wrong PC, can I stop it?
Yes. Because the Standard Restart uses a 60-second timer (/t 60), you have a one-minute window to fix the mistake. Simply run the tool again against the same PC and select the "Abort Restart" option. This sends a shutdown /a command, which instantly cancels the pending reboot.
