IT Pro Summary

This is a remote hardware diagnostic tool that leverages the native Windows powercfg utility over WinRM. It programmatically generates a detailed HTML report containing the last 14 days of battery usage, cycle counts, and degradation metrics on the target endpoint. It then securely transfers the file to the technician's local machine, opens it for immediate review, and deletes the remote payload to leave no trace.

When to Use This Tool

The Scenario

A remote user calls the help desk complaining that their laptop battery is draining too fast, not holding a charge, or shutting down unexpectedly at 30%. Traditionally, a technician would have to ask the user to bring the laptop into the office for hardware testing, or walk the user through opening an elevated command prompt, generating the report, finding the hidden HTML file, and emailing it to the help desk.

The Fix

While the user is explaining the issue on the phone, the technician clicks "Battery Rpt" in the UHDC. Within 5 seconds, the script generates the report remotely, pulls it across the network, and opens it in the technician's web browser. The technician can immediately compare the "Design Capacity" against the "Full Charge Capacity" to definitively tell the user, "Your battery has degraded to 40% of its original capacity. I am ordering a replacement part right now."

Interactive Training Engine

When a junior technician runs this tool with Training Mode enabled, the UHDC pauses execution at each critical phase. It displays a pop-up explaining why the action is necessary, what the in-person equivalent would be, and reveals the exact PowerShell code being executed under the hood. Here is what the trainee sees during a Battery Report generation:

[TRAINING] STEP 1: GENERATE REMOTE BATTERY REPORT

WHEN TO USE THIS:
Use this when a user complains that their laptop battery is draining too fast, not holding a charge, or shutting down unexpectedly.

WHAT IT DOES:
We are using WinRM to execute the native Windows 'powercfg' utility on the remote machine. This generates a detailed HTML report containing the last 14 days of battery usage, cycle counts, and degradation metrics, saving it to a temporary folder.

IN-PERSON EQUIVALENT:
Open an elevated Command Prompt on the user's PC and type 'powercfg /batteryreport /duration 14'.

Invoke-Command -ComputerName $Target -ScriptBlock { cmd.exe /c "powercfg /batteryreport /output C:\Temp\battery-report.html /duration 14" }
[TRAINING] STEP 2: RETRIEVE AND DISPLAY REPORT

WHEN TO USE THIS:
This step brings the generated report back to your workstation so you can analyze the 'Design Capacity' versus the 'Full Charge Capacity' to determine if the battery needs physical replacement.

WHAT IT DOES:
We are copying the HTML file over the network (via the C$ administrative share) to your local Temp folder, opening it in your default web browser, and then deleting the remote copy to leave no trace.

IN-PERSON EQUIVALENT:
Navigate to the folder where the report was saved, double-click the HTML file to open it in Edge/Chrome, and review the battery health statistics.

Copy-Item '\\$Target\C$\Temp\battery-report.html' -Destination '$localPath'
Start-Process '$localPath'
Remove-Item '\\$Target\C$\Temp\battery-report.html'

Technical Execution Pipeline

  1. Pre-Execution Validation: Executes a fast ICMP Ping to verify endpoint connectivity before attempting WinRM connections.
  2. Remote Execution (WinRM): Establishes a remote PowerShell session over Port 5985 (HTTP). It ensures the C:\Temp directory exists, then executes the powercfg command. It wraps the command in cmd.exe /c to ensure the legacy executable's arguments are parsed perfectly by the remote host.
  3. File Retrieval (SMB): Bypasses WinRM to connect to the endpoint's hidden C$ administrative share over Port 445 (SMB). It copies the newly generated HTML file directly to the technician's local $env:TEMP directory.
  4. Local Rendering: Uses Start-Process to hand the HTML file off to the technician's default web browser (e.g., Chrome or Edge) for immediate viewing.
  5. Forensic Cleanup: Re-uses the SMB connection to silently delete the HTML file from the remote user's C:\Temp drive, ensuring no clutter is left behind.

Q&A

Q: What exactly should I be looking for in the HTML report?
Scroll down to the "Installed batteries" section. Look at the Design Capacity (what the battery held when it was brand new) and compare it to the Full Charge Capacity (what it holds right now). If the Full Charge Capacity is significantly lower (e.g., 50% or less of the Design Capacity), the battery is physically degraded and should be replaced.
Q: Why is the `powercfg` command wrapped inside a `cmd.exe /c` call?
powercfg.exe is a legacy DOS executable. When passing complex arguments (like file paths with slashes) through a remote PowerShell Invoke-Command block, PowerShell's parser can sometimes mangle the string. Wrapping it in cmd.exe /c forces the remote machine to execute the command exactly as it would in a standard command prompt, guaranteeing success.
Q: What happens if I run this tool against a desktop computer instead of a laptop?
The WinRM connection will succeed, but the powercfg command will fail to generate a report because desktop motherboards lack the necessary battery API telemetry. The script handles this gracefully by detecting that the file was not created and alerting the technician in the console.
